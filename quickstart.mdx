This guide walks you through installing rrobot, loading example data, computing robust distances, and flagging outliers. The example uses the Hawkins-Bradu-Kass (`hbk`) dataset from robustbase — a classic outlier detection benchmark that ships with rrobot's dependencies.

<Steps>

  <Step title="Install rrobot">
    Install the released version from CRAN, or install the development version directly from GitHub.

    <CodeGroup>

    ```r CRAN
    install.packages("rrobot")
    ```

    ```r GitHub
    # Requires the remotes package
    install.packages("remotes")
    remotes::install_github("mandymejia/rrobot")
    ```

    </CodeGroup>

  </Step>

  <Step title="Load the package">
    ```r
    library(rrobot)
    ```
  </Step>

  <Step title="Load example data and compute robust distances">
    The `hbk` dataset contains 75 observations and 4 columns. The first 14 rows are known outliers. Load the data, extract the predictor columns, and compute squared robust distances using `compute_RD()`.

    ```r
    # Load the Hawkins-Bradu-Kass benchmark dataset
    data("hbk", package = "robustbase")

    # Use the three predictor columns (X1, X2, X3)
    X <- as.matrix(hbk[, c("X1", "X2", "X3")])

    # Compute robust distances via MCD (mode = "auto" selects the best subset)
    rd <- compute_RD(X, mode = "auto", dist = TRUE)

    # rd$RD        — squared robust distances for all 75 observations
    # rd$ind_incld — row indices used to estimate the robust covariance
    # rd$ind_excld — row indices excluded from the covariance estimate
    # rd$xbar_star — robust mean vector (length 3)
    # rd$S_star    — robust covariance matrix (3 × 3)
    ```

    <Note>
      `compute_RD()` calls `covMcd()` internally to identify the subset of observations with the smallest covariance determinant. This makes the robust mean and covariance resistant to masking by outliers.
    </Note>
  </Step>

  <Step title="Detect outliers with threshold_RD()">
    Pass the original data matrix and the `RD_result` object from the previous step. The `SI_boot` method (the default) bootstraps a threshold from single-imputed distances and is a reliable choice for most datasets.

    ```r
    result <- threshold_RD(
      x       = X,
      method  = "SI_boot",
      RD_obj  = rd,
      alpha   = 0.01,   # significance level for the threshold
      B       = 1000,   # number of bootstrap resamples
      boot_quant = 0.95 # bootstrap quantile used to set the threshold
    )
    ```
  </Step>

  <Step title="Inspect the results">
    The returned object is of class `RD`. Access flagged outliers and the threshold value through the `$thresholds` slot.

    ```r
    # Row indices of observations flagged as outliers
    result$thresholds$flagged_outliers

    # The numeric threshold applied to the robust distances
    result$thresholds$threshold

    # The full robust distance object (same as rd above)
    result$RD_obj$RD
    ```

    For the `hbk` dataset, `flagged_outliers` should recover observations near rows 1–14, which are the ground-truth outliers planted in this benchmark.
  </Step>

  <Step title="Plot the results">
    The `plot()` method for class `RD` visualises robust distances alongside the threshold and highlights flagged observations.

    ```r
    plot(result)
    ```

    <Tip>
      The plot shows all robust distances as a scatter or histogram with a horizontal reference line at the threshold. Observations above the line are the flagged outliers.
    </Tip>
  </Step>

</Steps>

## What you have seen

The workflow above demonstrates the two-stage rrobot pipeline:

1. **`compute_RD()`** estimates a robust center and covariance via MCD, then returns squared Mahalanobis distances for every observation.
2. **`threshold_RD()`** performs univariate pre-screening, imputes detected univariate outliers, and applies a data-driven threshold to the robust distances.
